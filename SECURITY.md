# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **dfsync**, please report it responsibly.

Do **not** open a public GitHub issue for security vulnerabilities.

Instead, please report the issue privately to the project maintainers.

You may report vulnerabilities by:

- opening a **GitHub Security Advisory**
- contacting the maintainers privately if contact information is available

We will investigate all reports and respond as quickly as possible.

---

## Supported Versions

The **latest released version** of dfsync is currently supported.

Older versions may not receive security updates.

---

## Responsible Disclosure

Please allow maintainers time to investigate and fix the issue before publicly disclosing the vulnerability.

Responsible disclosure helps keep the ecosystem safe for everyone.
